# QuadFinder 4ペイン拡張仕様

- 文書種別: 製品・機能仕様
- 対象: macOS向けファイルマネージャ QuadFinder
- ステータス: Draft 1.0
- 基準日: 2026-07-11

## 1. 概要

QuadFinderは、一般的な1ペインおよびPath Finder型の2ペイン操作を維持しながら、1つのウィンドウを最大4つの独立したファイルブラウザへ分割できるmacOS向けファイルマネージャとする。

4ペイン時は、上段左・上段右・下段左・下段右からなる2×2グリッドを基本形とする。各ペインは独立した場所、履歴、表示形式、選択、タブを持ち、ペイン間でコピー、移動、比較、同期などの操作を行える。

## 2. 目的

- 最大4つの場所を同時に表示し、複数フォルダ間の整理作業を高速化する。
- Finderや2ペイン型ファイルマネージャから自然に移行できる操作体系を保つ。
- 画面を増やしても、コマンドの対象ペインを常に判別できるようにする。
- 4ペインを、単なる4ウィンドウではなく1つの保存可能なワークスペースとして扱う。
- キーボード、マウス、トラックパッド、アクセシビリティ操作を同等にサポートする。

## 3. 対象外

初期リリースでは次を対象外とする。

- 5ペイン以上への分割
- 任意位置への自由分割や入れ子状の分割
- 複数ウィンドウをまたぐ1つのグリッド
- macOSの「開く」「保存」システムダイアログの置換
- クラウドサービス固有の同期エンジンの実装

## 4. 用語

| 用語 | 定義 |
| --- | --- |
| ウィンドウ | 1つのQuadFinder作業空間 |
| ペイン | 独立してフォルダ内容を表示するファイルブラウザ |
| スロット | グリッド上の位置。TL、TR、BL、BRで表す |
| アクティブペイン | キーボード操作とペイン対象コマンドの既定対象 |
| ソースペイン | コピー、移動、比較などの入力側 |
| ターゲットペイン | コピー、移動、同期などの出力側 |
| ペインセット | ウィンドウ内のペイン構成、場所、タブ、表示設定を含む保存状態 |

スロット識別子は次の固定値を用いる。

```text
TL | TR
---+---
BL | BR
```

## 5. レイアウト仕様

### 5.1 ペイン数

ウィンドウは1〜4ペインを持つ。ペイン数はウィンドウ単位で変更でき、変更時に新規ウィンドウを生成しない。

| ペイン数 | 標準レイアウト | 説明 |
| --- | --- | --- |
| 1 | Single | 1ペインが全領域を使用 |
| 2 | Vertical / Horizontal | 左右分割または上下分割 |
| 3 | Leading / Trailing / Top / Bottom | 1ペインが一辺を占め、反対側を2分割 |
| 4 | Grid | 2×2グリッド |

### 5.2 2ペイン

- Vertical: 左右に並べる。初期値は50:50。
- Horizontal: 上下に並べる。初期値は50:50。
- ユーザーは分割方向を切り替えられる。

### 5.3 3ペイン

3ペインは空白セルを作らず、次の4形状から選択する。

- Leading: 左ペインが縦方向を占有し、右側を上下に分割
- Trailing: 右ペインが縦方向を占有し、左側を上下に分割
- Top: 上ペインが横方向を占有し、下側を左右に分割
- Bottom: 下ペインが横方向を占有し、上側を左右に分割

2ペインから3ペインへ増やす場合、アクティブペインを分割して新しいペインを作る。分割方向は現在の形状とポインタ位置を考慮して選ぶが、実行前のメニューで形状を明示する。

### 5.4 4ペイン

- TL、TR、BL、BRを用いる2×2グリッドとする。
- 中央の交点をドラッグすると、縦横の分割位置を同時に変更できる。
- 縦または横のディバイダだけをドラッグした場合、その軸の比率だけを変更する。
- ディバイダをダブルクリックすると、その軸を50:50へ戻す。
- Optionキーを押しながらダブルクリックすると縦横とも50:50へ戻す。

### 5.5 サイズ制約

- 各ペインの最小幅: 240 pt
- 各ペインの最小高さ: 180 pt
- 4ペインを維持できないウィンドウサイズでは、ディバイダ移動を制限する。
- ウィンドウ自体が最小要件を下回る場合、ペインを自動削除せず、内容領域を最小値まで縮小して警告表示する。
- フルスクリーン、Split View、外部ディスプレイ移動後もペイン数を維持する。

### 5.6 ペインの追加

「ペインを追加」は現在のペイン数が4未満のときだけ有効とする。

新規ペインの初期場所は、次の優先順位で決める。

1. アクティブペインで現在表示中のフォルダ
2. そのフォルダを開けない場合はユーザーのホームフォルダ
3. ワークスペーステンプレートに指定がある場合はテンプレートの場所

新規ペインは元ペインの表示形式を継承するが、履歴、選択、タブは独立して生成する。

### 5.7 ペインの削除

- アクティブペインを閉じた場合、残りのペインを自動的に再配置して空白を残さない。
- 閉じたペインに未完了のファイル操作がある場合でも、操作はウィンドウ単位のキューで継続する。
- 未保存のタブ状態は、直前に閉じたペインとして1回だけ復元可能とする。
- 最後の1ペインは閉じられない。「ペインを閉じる」は無効になる。

### 5.8 ペインの交換と最大化

- ペインヘッダを別のペインへドラッグすると位置を交換する。
- 「ペインを一時最大化」で、アクティブペインのみをウィンドウ全体に表示できる。
- 最大化中も他ペインは破棄せず、ファイル操作と監視を継続する。
- 同じ操作またはEscで元のグリッドに戻る。

## 6. ペインの状態モデル

各ペインは少なくとも次の状態を独立して保持する。

- 一意なペインID
- グリッド上のスロットまたは占有領域
- 現在のURL
- 戻る／進む履歴
- タブ一覧と選択中タブ
- 表示形式（アイコン、リスト、カラム）
- 並び順、グループ、表示列、アイコンサイズ
- 選択項目、フォーカス項目、スクロール位置
- 隠しファイルと拡張子の表示状態
- フィルタおよび検索条件
- 表示中のモジュールに対する参照コンテキスト

ウィンドウは次の共有状態を持つ。

- ペイン数とレイアウト形状
- アクティブペインID
- 直前アクティブペインID
- ソース／ターゲットの指定
- ディバイダ比率
- サイドバー、ツールバー、モジュール配置
- ファイル操作キュー
- ペインリンク設定

## 7. アクティブペイン

### 7.1 表示

- 常に1つだけアクティブペインが存在する。
- アクティブペインはアクセントカラーの1.5 pt枠と、選択状態のペインヘッダで示す。
- 非アクティブペインの選択項目は淡色表示し、選択自体は保持する。
- 色だけに依存せず、ヘッダの状態とアクセシビリティ属性でも識別可能にする。

### 7.2 切り替え

- ペイン内のクリックでそのペインをアクティブにする。
- Control+Tabで次のペイン、Control+Shift+Tabで前のペインへ移動する。
- Control+1〜4で画面上の読み順（TL、TR、BL、BR）に対応するペインを選択する。
- 方向移動はControl+Option+矢印キーで行う。
- ペイン切り替えだけでは選択、スクロール、タブを変更しない。

## 8. ナビゲーションとタブ

- 各ペインは複数タブを持てる。
- 新規タブ、タブを閉じる、戻る、進む、上位フォルダへ移動はアクティブペインだけを対象とする。
- タブを他ペインへドラッグすると移動する。Optionキー併用時は複製する。
- フォルダを「別ペインで開く」場合、最も近い非アクティブペインを使用する。
- 空きペインがなく、既存ペインを置換する可能性がある場合は、対象ペインをユーザーに示してから実行する。

## 9. ペイン間ファイル操作

### 9.1 ドラッグ＆ドロップ

- 同一ボリュームへの通常ドラッグは移動、別ボリュームへはコピーとするmacOS標準挙動を踏襲する。
- Optionでコピー、Commandで移動を明示できる。
- ドロップ先ペインを一時的に強調し、実行内容を「コピー」「移動」「リンク」で表示する。
- ペインの空白領域へのドロップは、そのペインで表示中のフォルダを宛先とする。
- フォルダ上で一定時間ホバーするとスプリングロードする。ただしアクティブペインは切り替えない。

### 9.2 コピー／移動コマンド

「他ペインへコピー」「他ペインへ移動」を提供する。

- 非アクティブペインが1つの場合は、そのペインを宛先として即時実行する。
- 非アクティブペインが2つ以上の場合は、方向付きの宛先選択UIを表示する。
- 宛先候補にはペイン番号、フォルダ名、パスを表示する。
- 直前に使用した宛先は既定候補になるが、確認なしに推測実行しない。

### 9.3 クリップボード

- Copy/Cut/PasteはmacOS標準に準拠する。
- Pasteはアクティブペインの現在フォルダを宛先とする。
- Cut元のペインを閉じてもクリップボード状態は保持する。

### 9.4 操作キュー

- コピー、移動、削除、圧縮、展開はウィンドウ共有の操作キューで管理する。
- 各ジョブにソースペイン、ターゲットペイン、開始時のURLを記録する。
- ペインの移動、閉鎖、ナビゲーションによって進行中ジョブの宛先を変更しない。
- 同一宛先に対する競合ジョブは順序制御し、競合ダイアログに元・先の完全パスを表示する。

## 10. ペインリンク

任意の2〜4ペインをリンクグループにできる。

初期リリースで提供するリンクモードは次の2つとする。

- 相対ナビゲーション: 1ペインで子フォルダへ移動したとき、リンク先も同名の子フォルダへ移動を試みる。
- 選択追従: 同名項目が存在する場合、リンク先でも対応項目を選択する。

リンク先に対応する場所または項目がない場合、そのペインは移動せず、ヘッダに非侵襲的な通知を表示する。リンク操作によるファイル変更は行わない。

## 11. 比較・同期

- 2つのペインをソースとターゲットに指定し、フォルダ比較を実行できる。
- 3ペイン以上がある場合、比較対象はアクティブペインとユーザーが選択した1ペインとする。
- 比較結果はファイル名、サイズ、更新日時、必要に応じてチェックサムで分類する。
- 同期はプレビューを必須とし、作成、上書き、削除件数を実行前に表示する。
- 双方向同期、片方向ミラー、欠落項目のみコピーをモードとして区別する。
- 削除を伴う同期はUndo可能な範囲を明示し、ゴミ箱を経由できない対象では追加確認する。

## 12. サイドバー、ツールバー、モジュール

### 12.1 サイドバー

- サイドバーは既定でウィンドウ共有とする。
- サイドバーで場所をクリックするとアクティブペインに開く。
- 場所を特定ペインへドラッグすると、そのペインで開く。
- Option+クリックで直前アクティブだったペインに開く。

### 12.2 ツールバー

- 戻る、進む、表示切替、新規タブなどのナビゲーション操作はアクティブペイン対象。
- ペイン数、レイアウト、ペイン最大化、ペインリンクはウィンドウ対象。
- 破壊的操作のツールチップには対象ペイン名を表示する。

### 12.3 モジュール

モジュールは参照コンテキストを次のいずれかに設定できる。

- Active: アクティブペインに追従
- Pinned: 指定ペインに固定
- Pair: 指定した2ペインを参照（比較、同期など）
- Window: ウィンドウ全体を参照（操作キューなど）

プレビュー、情報、属性、ターミナルは既定でActive、操作キューはWindow、比較と同期はPairとする。

## 13. メニューと標準ショートカット

| 操作 | 既定ショートカット |
| --- | --- |
| ペインを追加 | Control+Option+Command+N |
| アクティブペインを閉じる | Control+Option+Command+W |
| 次／前のペイン | Control+Tab / Control+Shift+Tab |
| ペイン1〜4へ移動 | Control+1〜4 |
| 方向でペイン移動 | Control+Option+矢印 |
| アクティブペインを一時最大化 | Control+Option+Return |
| ペイン位置を交換 | メニューから実行。初期値は未割当 |
| 他ペインへコピー | F5 |
| 他ペインへ移動 | F6 |
| 新規フォルダ | Shift+Command+N |

ショートカットは設定画面で変更可能とし、macOSおよび各シェルの予約操作との衝突を警告する。

## 14. コンテキストメニュー

フォルダおよびタブに次の項目を追加する。

- 新しいペインで開く
- ペイン1〜4で開く
- アクティブペインを分割
- このペインを閉じる
- このペインを一時最大化
- ペイン位置を交換
- 他ペインへコピー
- 他ペインへ移動
- 比較対象に指定
- 同期元／同期先に指定

存在しないペイン番号や、実行不能な操作は非表示ではなく無効表示し、理由をツールチップまたはヘルプタグで示す。

## 15. 状態保存と復元

### 15.1 自動保存

アプリ終了、ウィンドウ終了、レイアウト変更時に次を保存する。

- ペイン数と形状
- ペインの位置、サイズ比率、最大化状態
- 各ペインのタブと現在URL
- 各タブの表示設定
- アクティブペイン
- ペインリンク
- サイドバー、ツールバー、モジュール配置

### 15.2 復元不能な場所

外部ボリューム、ネットワーク共有、クラウド項目などを復元できない場合は、当該タブにプレースホルダを表示し、再接続または別の場所を選べるようにする。他のペインの復元は継続する。

### 15.3 ペインセット

ユーザーは現在の構成を名前付きペインセットとして保存できる。ペインセットには資格情報やセキュリティスコープ外のアクセス権を保存しない。

## 16. Undo、エラー、確認

- ファイル操作のUndo/Redoはウィンドウ単位の時系列で管理する。
- Undo対象には、実行元ペインではなく元URLと先URLを記録する。
- ペインを閉じてもUndo履歴を失わない。
- エラー通知は該当ジョブとペインに関連付けるが、モーダルダイアログの乱立を避ける。
- 削除、上書き、同期など複数項目へ影響する操作では、対象件数と完全パスを確認できるようにする。
- 読み取り専用、権限不足、オフライン、容量不足は項目別に再試行またはスキップできる。

## 17. アクセシビリティ

- VoiceOverの読み順は、ウィンドウ共通UIの後にTL、TR、BL、BRとする。
- 各ペインは「ペイン1、Downloads、アクティブ」のようなラベルを持つ。
- ディバイダはキーボードで1 pt単位、Shift併用で10 pt単位に調整可能とする。
- アクティブ状態、コピー先、比較対象は色以外の枠、アイコン、ラベルでも示す。
- Reduce Motion有効時は、分割・最大化アニメーションを短縮または無効化する。
- Full Keyboard Access、Voice Control、Switch Controlに対応する。

## 18. パフォーマンス要件

- 4つのローカルフォルダを開いた通常状態で、ペイン切り替えの視覚応答を100 ms以内とする。
- レイアウト変更は、ファイル列挙を再実行せず250 ms以内に完了することを目標とする。
- 非表示または一時最大化で隠れたペインは、監視を維持しつつサムネイル生成を抑制する。
- ファイル監視、Quick Look、フォルダサイズ計算はペイン単位で重複排除する。
- 大規模フォルダの読み込みはUIをブロックせず、各ペインで独立してキャンセル可能とする。
- 1ペインの読み込み失敗やネットワーク遅延が、他ペインの操作を停止させてはならない。

## 19. セキュリティとmacOS連携

- App Sandbox、ユーザー選択ファイル、security-scoped bookmarkを前提に設計する。
- 各ペインが参照する場所へのアクセス権を独立して検証する。
- 権限のないペインはエラー状態にし、他ペインのアクセスを巻き戻さない。
- iCloud Drive、Dropbox、Google Drive、OneDrive等はFile Providerまたはローカル公開領域を通して扱う。
- クラウド専用項目は、コピーや比較に必要な場合にダウンロード状態を明示する。
- FinderをOSから削除せず、QuadFinderを主ファイルマネージャとして使う設定は任意とする。

## 20. 設定項目

「設定 > ペイン」に次を設ける。

- 起動時のペイン数とレイアウト
- 2ペインの既定分割方向
- 3ペインの既定形状
- 新規ペインの初期場所
- アクティブペインの枠色と強調度
- ペイン番号の常時表示
- ディバイダ比率の記憶
- ペインごと／全ペイン共通の表示設定
- ドラッグ時のコピー／移動確認
- ペイン切り替えショートカット
- アプリ終了時のペインセット復元

## 21. 受け入れ基準

### AC-01: 4ペイン生成

1ペインから「ペインを追加」を3回実行すると、同一ウィンドウ内に4つの操作可能なペインが2×2で表示される。5回目の追加操作は実行できない。

### AC-02: 独立ナビゲーション

4ペインが異なるフォルダ、タブ、表示形式、選択、スクロール位置を保持し、1ペインの操作が他ペインを意図せず変更しない。

### AC-03: 明確な対象

どの時点でもアクティブペインが1つだけ明示され、戻る、進む、新規タブ、Paste、削除がそのペインに作用する。

### AC-04: ペイン間コピー

任意のペインから別の任意のペインへドラッグまたはコマンドでコピーでき、3つの候補がある場合は宛先を識別して選べる。

### AC-05: レイアウト変更

1〜4ペイン間を切り替えても、残存ペインのタブ、履歴、選択を維持する。閉じた直後のペインは1回復元できる。

### AC-06: サイズ変更

4ペインの中央交点、縦ディバイダ、横ディバイダを操作でき、最小サイズ制約を超えて縮小できない。

### AC-07: 保存復元

4ペインの場所、タブ、表示、分割比率を保存し、アプリ再起動後に同じ構成を復元できる。アクセス不能な場所が1つあっても他の3ペインは復元される。

### AC-08: 最大化

任意ペインを一時最大化し、元の4ペイン構成、処理中ジョブ、選択状態を失わず復帰できる。

### AC-09: ファイル操作の安定性

操作開始後にソースまたはターゲットのペインを閉じたり別フォルダへ移動しても、ジョブは開始時のURLを用いて正しく完了する。

### AC-10: アクセシビリティ

マウスを使わず4ペイン間を移動し、ディバイダ調整、フォルダ移動、コピー先選択ができる。VoiceOverがペイン番号、場所、アクティブ状態を読み上げる。

## 23. 設計上の原則

4ペイン化で最も重要なのは、「現在どのペインを操作しているか」と「ファイルがどこからどこへ移動するか」を常に明示することである。ペイン数が増えても、対象を推測して破壊的操作を実行してはならない。

内部実装では、表示上のスロットとペインの永続IDを分離する。ペインを交換しても履歴や処理中ジョブはペインIDに追従し、画面位置だけが変わるモデルとする。
